SQL Server - Debug a SQL Stored procedure


When a stored procedure is not producing the expected result, sometimes it is very difficult to identify the rootcause of the issue. The SQL procedure can be debugged as like a .Net program. The debugging process is recommended directly on the server, if possible. Follow the below steps to debug a SQL stored procedure.

Login directly onto the SQL server using a Windows account. If there is no windows account, inform the System Admin or Database Admin to create a new account.

After Login, 'Debug' the query in SQL management studio by clicking the "Debug" icon, setting the breakpoints wherever necessary. You can step into stored procedure using F11 key.
